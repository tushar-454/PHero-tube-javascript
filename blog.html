1. Discuss the scope of var, let, and const

Note the following codes first

if (true) {
  var varVariable = 'This is var';
}
console.log(varVariable); // output - 'This is var'
if (true) {
  let letVariable = 'This is let';
}
console.log(letVariable); // output - Reference error

Here is a block scope taken at 1st. There a variable is taken with var and console.log(varVariable) is done.

Here is a block scope taken at 2nd. There a variable is taken with let and console.log(letVariable) is done.

But the output of the two is completely different. It gives variable output with var and let gives error. A subtle reason for this is scope. var is function scope and let is block scope. Since if block scope, var can be used from outside but since let is block scope, we cannot use it outside if block.

If we want to get let as output then console.log() should be done in its block.

if (true) {
  var varVariable = 'This is var';
}

console.log(varVariable); // output - 'This is var'

if (true) {
  let letVariable = 'This is let';
  console.log(letVariable); // output - 'This is let'
}

We can define multiples of var but not let.

if (true) {
  var varVariable = 'This is var';
  var varVariable = 'This is var 2'; // we can do it.
}

console.log(varVariable); 
/* output - 'This is var 2'. But we can't get the output as per our requirement and see our output changed.*/

if (true) {
  let letVariable = 'This is let';
  let letVariable = 'This is let 2';
  console.log(letVariable);
  /* output - error - I will get error here because we cannot define variable with multiple time let with same name. But I can change it in the same block.*/
}
if (true) {
  let letVariable = 'This is let';
  letVariable = 'This is let 2';
  console.log(letVariable); // output - 'This is let 2'
}
/* See here again we used `letVariable` in two places, what is surprising, now it cannot be used in the name, now what happened. Because let block scope they are in different blocks so it can be done.*/

I can't redefine to const and reassign is out of the question

if (true) {
  const constVariable = 'This is let';
  const constVariable = 'This is let 2';
  console.log(constVariable); 
  /* output - error - I will get error here  because I can't redefine const and reassign  doesn't come in the question. */
}

But if the value of const variable is Array, Object instead of String, number then we cannot redefine but we can update the data. 

if (true) {
  const constVariable = {
    name: 'JS',
    age: '25 years',
  };
  constVariable.name = 'JavaScript';
  console.log(constVariable); // output - ✔
}
// But whatever const or let, I can't do anything outside of its block. error will occur.
constVariable.name = 'Scripting Language'; // output - error ❌
console.log(constVariable); // output - error ❌
